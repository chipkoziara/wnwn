Ensure long lists remain navigable and visible within viewport constraints.

## Acceptance Criteria

- Long lists remain navigable without content being cut off/unreachable.
- Scrolling behavior is consistent across inbox/actions/projects/view results.
- Add regression checks for viewport height + cursor visibility.

## Resolution (2026-03-15)

Implemented shared scroll handling for list-style views using a `scrollOffset` plus `visibleRange()`/`ensureScroll()` helpers. Inbox/actions, project list/detail, saved views, view results, and weekly review now render only the visible slice for the current terminal height and automatically keep the active cursor in view, including after window resize events.
