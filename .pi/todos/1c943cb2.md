Ensure overlays and sub-components do not render duplicate command help rows, with the app-level helper remaining the single contextual source of truth.

## Acceptance Criteria

- Remove duplicate/help footer rendering from overlays or sub-components where the parent app already renders contextual help.
- App-level help text remains accurate for the active context (including date picker and other overlays).
- No screens should show two competing rows of command hints.
- Document the pattern in STATUS.md if a reusable convention is introduced.

## Resolution (2026-03-15)

Removed the date picker's internal footer help row so the app-level `helpText()` bar remains the only command-hint source while the overlay is active. The date picker still renders its contextual content/help copy near the time toggle, but command guidance now appears once and stays consistent with the current app mode.
