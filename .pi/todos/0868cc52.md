Investigate and fix key routing conflicts where typing `u` is blocked by undo behavior.

## Acceptance Criteria

- Reproduce and identify the exact views/modes where `u` is swallowed.
- Ensure text inputs can always type literal `u` unless an explicit modal shortcut should take precedence.
- Preserve undo behavior where intended and document key-routing rules.

## Dedupe note

This item remains canonical for the specific **`u` key input-routing bug**.

## Resolution (2026-03-15)

Fixed by disabling Bubble textinput's default suggestion-accept key binding during TUI model initialization. That binding also used `u`, which caused inline editors to swallow literal `u` keypresses in some views. App-level undo remains active only in `modeNormal`, so text-entry flows now accept `u` normally without regressing undo behavior.
