Add command-line query and mutation commands so users and agents can inspect and update GTD data without launching the TUI.

## Acceptance Criteria

- Add CLI commands to search/query tasks and projects from the terminal.
- Add CLI commands to update task and/or project records by stable identifier.
- Support structured query DSL filters and project/task matching at minimum.
- Support at minimum these mutations where appropriate: state, deadline, scheduled, tags, text/title, notes, URL, waiting-on.
- Support clearing fields where the TUI supports clearing them.
- Output should be stable and script-friendly for human and agent/tool workflows.
- Document examples for terminal usage and agent-oriented workflows.

## Notes

This unifies the earlier CLI search and CLI mutation ideas into one agent-friendly command surface. Together these capabilities would allow local automation/coding agents to inspect GTD state, find matching records, and apply structured updates from shell workflows.

## Resolution (2026-03-15)

Added `wnwn query` and `wnwn update` commands to `cmd/wnwn/main.go`. `query` supports script-friendly JSON output over tasks/projects using the same DSL as Views, including archived-task inclusion for task queries. `update` supports task/project mutation by stable ID for state, deadline, scheduled, tags, text/title, notes, URL, waiting-on, and explicit field clearing. README and STATUS.md were updated with usage examples oriented toward terminal and agent workflows.
