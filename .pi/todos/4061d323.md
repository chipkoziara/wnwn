## Overview
Add export functionality and backup support to wnwn.

## Export Formats
- Export tasks/projects to JSON (primary target)
- Potentially CSV, plain text, or other formats
- Consider exporting full data set or filtered views (e.g. by project, list, date range)

## Backup
- Backup the markdown data directory (WNWN_DATA_DIR) to a timestamped archive
- Could be a CLI subcommand: `wnwn backup` or `wnwn export --format json`
- Consider auto-backup before destructive operations

## Notes
- Service layer is stateless (read → modify → write), so export logic fits naturally in a new service method or CLI command
- Markdown is already the canonical format; export is a projection of it
- JSON schema should mirror the internal model types (Task, TaskList, Project, SubGroup)
