Add minimal date range syntax to the query DSL instead of introducing a broader macro language.

## Acceptance Criteria

- Support inclusive date range syntax for date fields using existing DSL concepts, e.g. `deadline:today..7d`, `scheduled:2026-03-01..2026-03-07`.
- Reuse existing relative date tokens (`today`, `tomorrow`, `Nd`) and fielded query structure.
- Preserve current comparator syntax (`<`, `<=`, `>`, `>=`) and boolean composition.
- Document how common concepts map onto the DSL without adding new named macro terms:
  - overdue → `deadline:<today`
  - due today → `deadline:today`
  - due this week → `deadline:today..7d`
  - scheduled this week → `scheduled:today..7d`
  - before today across multiple fields → `deadline:<today OR scheduled:<today`
- Add parser and matcher tests for inclusive range behavior and relative token boundaries.
